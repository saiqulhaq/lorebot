lorebot is **~7 source files by design**. There's no framework, no plugin system, no build step — Bun runs the TypeScript directly. This page and the next two map the codebase for contributors.

## Two roles, one package

lorebot ships two independent roles in a single codebase: the **Slack Q&A bot** (`src/*.ts`) and the **graphify sync orchestrator** (`src/sync/*.ts`), which builds merged knowledge graphs and pushes them into app repos. They share config loading and logging, nothing else — sync can run standalone via `bun run sync`, and a sync failure can never take down Slack answering. See [Graphify Sync](/docs/sync) for the deep dive.

## Repository layout

```text
lorebot/
├── src/                  # the bot (see Modules)
├── lorebot.config.json   # behavior config, hot-reloaded
├── agent/opencode.jsonc  # default_agent marker installed into the KB clone
├── fixtures/kb/          # tiny fake KB for local smoke tests
├── test/                 # bun test suites (one per module)
├── docs/                 # this documentation site (Mintlify)
├── .env.example          # connection settings template
└── Dockerfile
```

## Design principles

- **Config over code** — anything a non-developer might want to change belongs in `lorebot.config.json`, not TypeScript
- **Fail closed, degrade gracefully** — permission gates silently ignore rather than error; a broken config reload keeps the previous config; KB sync failures never crash the bot
- **No hidden state** — thread↔session mapping is one sqlite table; everything else lives server-side in OpenCode or in git
- **Privacy by default** — Q&A text only reaches logs at `debug` level
